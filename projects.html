Data Engineering - 2026
Toronto Airbnb ETL Pipeline

A containerized end to end data pipeline built on real Toronto Airbnb listings data. Raw data is extracted, cleaned and transformed with Python and Pandas, warehoused in a PostgreSQL star schema, and surfaced through an interactive Streamlit dashboard. Workflow is orchestrated via Apache Airflow DAGs with automated data quality validation. Inspired by a university data engineering project.

View on GitHub
Pipeline architecture
Extractor loads raw listings into a staging schema · Transformer cleans, type-casts, and normalises data into a star schema (fact_listings + 3 dims) · Airflow DAG schedules weekly runs with extract => transform => validate => branch tasks · Data quality gate enforces ≥70% row retention before marking a run successful
Dashboard insights
15,776 listings across 140 Toronto neighbourhoods · Top neighbourhoods by listing count · Room type distribution · Availability trends · Top hosts by review volume · Reviews per month by room type and neighbourhood
Status
Complete - fully containerized, single command deployment via Docker Compose.

Full-Stack - 2025
Coach-Fit

Your training, quantified. Most fitness apps just store numbers - Coach-Fit treats every logged workout as structured data and builds on top of it. Each session's exercises, sets, reps, weight, and RPE feed a running history that powers trend charts, muscle-balance breakdowns, and recovery insights, so progress is something you can actually see instead of guess at. A Grok layer reads that history to suggest what to train next. Built full-stack with secure authentication and deployed live, it's really a personal data pipeline dressed as a fitness tracker - data-centric by design, and completely free to use.

Try it live View on GitHub
What it tracks
Every set, rep, weight, and RPE becomes structured data · Weekly volume and trend charts · Muscle-balance distribution · Recovery flags from training load · Consistency and streak scoring across sessions.
AI layer
Grok reads each user's logged history server-side to generate personalised training recommendations, progression suggestions, and rebalancing cues.
Built with
React + TypeScript frontend · Node.js / Express API · MongoDB for flexible workout documents · JWT authentication · deployed on Vercel.
Status
Live - free to try, with new tracking features in progress.
