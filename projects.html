Data Engineering - 2026
Toronto Airbnb ETL Pipeline

A containerized end to end data pipeline built on real Toronto Airbnb listings data. Raw data is extracted, cleaned and transformed with Python and Pandas, warehoused in a PostgreSQL star schema, and surfaced through an interactive Streamlit dashboard. Workflow is orchestrated via Apache Airflow DAGs with automated data quality validation. Inspired by a university data engineering project.

View on GitHub
Pipeline architecture
Extractor loads raw listings into a staging schema · Transformer cleans, type-casts, and normalises data into a star schema (fact_listings + 3 dims) · Airflow DAG schedules weekly runs with extract => transform => validate => branch tasks · Data quality gate enforces ≥70% row retention before marking a run successful
Dashboard insights
15,776 listings across 140 Toronto neighbourhoods · Top neighbourhoods by listing count · Room type distribution · Availability trends · Top hosts by review volume · Reviews per month by room type and neighbourhood
Status
Complete - fully containerized, single command deployment via Docker Compose.

Power BI dashboard analyzing 58,000+ TTC bus delay incidents across Toronto in 2025. Uncovers patterns in delay causes, affected routes, time of week trends, and directional breakdowns using City of Toronto open data. Built with Power Query for data cleaning and transformation.

View on GitHub
Dashboard visuals
